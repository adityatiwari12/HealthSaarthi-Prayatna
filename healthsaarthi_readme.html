The Crisis
India's emergency healthcare operates blind
Not because resources don't exist — but because they are invisible to each other. Ambulances drive blind. Hospitals prepare nothing. Patients pay with their lives.

Information Silo

Ambulance has zero visibility into ICU availability, ventilators, specialists on duty across hospitals in real time.

Identity Gap

Unconscious patient arrives with zero medical history — no blood type, no allergies, no medications, no conditions known.

Coordination Void

No city-level, real-time view of healthcare resources exists for coordinators. Each hospital is a silo.

The Solution
End-to-end patient journey
From stranger finding an unconscious person to hospital fully prepared — in under 12 minutes. Zero phone calls. Zero paperwork. Zero information lost.
T + 0s
Patient found — tap on lock screen
Any bystander taps the always-on emergency display. No unlock needed. AI agent captures GPS and simultaneously calls all registered family members.
T + 8s
Doctor scans QR — full history in 3 seconds
Blood type, allergies, conditions, medications, last ECG. No intake form. No guessing. AES-256 encrypted, 15-minute token expiry, doctor-verified access only.
T + 15s
IoT patch applied — live vitals stream begins
ESP32 + MAX30102 connects via GSM. HR, SpO₂, BP streaming to hospital dashboard in real time. Works on 2G. No WiFi needed.
T + 20s
AI dispatch engine fires — optimal hospital selected
Scores all hospitals simultaneously: ICU availability, specialist on duty, OSRM ETA, stress index. Conflict resolution if two ambulances need the same last bed.
T + 30s
Hospital pre-arms — before ambulance moves
ICU bed reserved, cardiologist alerted, blood stock confirmed, preliminary tests ordered. Hospital receives everything before the ambulance even leaves the scene.
T + 12 min
Arrival — zero delay, zero paperwork
Geofence trigger at 500m: bed flips to Occupied, full vitals journey attaches to EMR, family notified with room number. Doctor walks in knowing everything.

Intelligence Core
AI dispatch algorithm
Not the nearest hospital. The right hospital. A multi-variable weighted scoring engine with dynamic weights that shift based on patient triage severity.
Triage score — computed from live vitals
# Starts at 100, deductions per danger sign def compute_triage(vitals, patient): score = 100 # SpO₂ — highest weight if vitals.spo2 < 90: score -= 35 # CRITICAL elif vitals.spo2 < 94: score -= 20 elif vitals.spo2 < 97: score -= 8 # Heart Rate (normal 60–100 bpm) if vitals.hr > 130: score -= 25 # Tachycardia elif vitals.hr < 50: score -= 25 # Bradycardia # Blood Pressure systolic if vitals.bp_sys < 80: score -= 25 # Shock if vitals.bp_sys > 180:score -= 20 # Crisis # Pre-existing (from QR) if 'cardiac' in patient.conditions: score -= 10 if patient.age > 70: score -= 8 return max(0, min(100, score))
Dynamic scoring weights by triage level
ParameterCRITICALSERIOUSMODERATE
ICU Availability30%25%20%
Specialist Match20%25%30%
OSRM ETA30%20%15%
Hospital Capacity10%20%25%
Ventilator10%10%10%
When CRITICAL: ETA weight doubles — getting there fast is life-or-death. When STABLE: specialist precision matters more than speed. Weights shift automatically.
